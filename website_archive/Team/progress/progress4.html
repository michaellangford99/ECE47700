Description of design efforts:

I spent most of this week mainly working on the Software Overview, revising the project proposal, and working on coding UART communication between a Raspberry Pi and an STM32F446, as well as starting to write the 3D point cloud to 2D occupancy grid algorithm.
I was writing the UART communication as shown in the first source as well as the figure shown below. This is mainly for the navigation computer to be able to send flight instructions to the microcontroller and for the microcontroller to send sensor data to the navigation single board computer.
Figure 1: A diagram of the UART protocol (PC is the Raspberry Pi)
I also was programming the 3D point cloud to 2D occupancy grid algorithm using the sources shown below. I have never coded an algorithm like this before, so I have done research into multiple methods of doing so as shown in the sources 2-5. I am working on the different versions of this algorithm as previously mentioned to see which version works the most optimally for obstacle aversion and the speed of running so on the Raspberry Pi.
